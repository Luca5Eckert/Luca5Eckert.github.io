// query flow — HippoRAG 2 retrieval
  • HippoRAG 2 retrieval — neurobiologically inspired: pgvector finds anchor nodes (like sensory cortex), Neo4j runs Personalized PageRank from those seeds (like hippocampal indexing), RRF fuses both candidate sets. Surfaces multi-hop connections — a query about "weight updates" reaches "backpropagation" via "gradient descent" even if the query vector isn't close to any single source
  • Dual ingest pipelines running in parallel — semantic: chunker (200 tok, 40 overlap) → ONNX all-MiniLM-L6-v2 in-JVM → pgvector; graph: Gemini Flash OpenIE → Neo4j typed triples (phrase nodes, passage nodes, context edges) + automatic synonymy edges where cosine similarity exceeds threshold
  • No Python sidecar — ONNX Runtime + DJL HuggingFace Tokenizers run all-MiniLM-L6-v2 directly on the JVM. The model is a file, not a service. Eliminates cross-process overhead and keeps the architecture honest: one runtime, no hidden coupling
  • Hexagonal architecture — every external dependency (pgvector, Neo4j, Gemini Flash, ONNX) is accessed through a port interface. Migrating concept extraction from Gemini Flash to a local Ollama model requires changing one adapter, not the application layer
  • Graph View with D3.js — interactive force-directed visualization of the concept graph; node size by degree centrality; clicking a concept shows all sources referencing it and an AI-generated synthesis grounded exclusively in ingested content, never in model world knowledge
  • Synonymy edges — concept nodes whose embedding vectors exceed a cosine similarity threshold are automatically linked, so backprop and backpropagation are connected without manual normalization. The graph normalizes itself as it grows

  • Node-RED as IoT orchestrator — bridges MQTT broker to Spring backend; on entry subscribes to device events and calls POST /access/entry; on gate release receives POST /open-gate and publishes back to ESP32 via MQTT
  • Two DDD bounded contexts — access (entry/exit lifecycle, gate coordination) and payment (Mercado Pago Checkout Pro, webhook confirmation, SSE stream); each with 4-layer clean architecture
  • WebFlux SSE for payment status — backend holds a reactive SSE connection per payment ID and pushes paid: true the moment the Mercado Pago webhook lands; no polling, unidirectional, stateless from server side
